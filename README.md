# Scientist-Skill：多科学家研究品味评估系统

> **集成 20+ 位科学家的研究品味，从古早大师到诺奖级到普通PI**
> 支持个人评估、群体讨论、跨级别对比三种模式

[![License: MIT](https://img.shields.io/badge/license-MIT-green.svg)]()

## 这是什么？

一个**元级别**的科学品味评估系统，内置 20+ 位科学家的品味画像，覆盖三个层级：

| 层级 | 代表人物 | 特点 |
|------|---------|------|
| **古早大师** | 爱因斯坦, 费曼, 牛顿, 孟德尔, 伽利略 | 第一性原理思维，开创性方法论 |
| **诺奖级** | 杜德纳, 辛顿, 卡里科, 彭罗斯, 山中伸弥 | 前沿方法论，范式转换 |
| **通常PI** | 10种代表性PI原型 | 实用导向，出版驱动 |

## 四种使用模式

### 模式1：单人品味
"用爱因斯坦的品味评估我的项目"

### 模式2：层级选择
"用诺奖级科学家的标准评估这个方向"

### 模式3：群体讨论
"召开一个品味评审会来讨论我的项目" — 多位科学家各抒己见，最后综合

### 模式4：对比分析
"比较古早大师和诺奖级对这个问题的看法"

## 示例：群体讨论模式

**输入：** "我想用图神经网络预测新材料的性质"

```
Einstein: +0.35 — 能否揭示统一原理？参数太多不够简约。
Feynman:  +0.45 — 先算一个具体的数字。然后去实验验证。
Newton:   +0.40 — 精度具体是多少？误差棒有多大？
Doudna:   +0.50 — 能直接指导实验合成新材料吗？
Karikó:   +0.55 — 有坚实的初步结果吗？别人不看好反而可能是好方向。
Hinton:   +0.65 — 图神经网络能学到好的材料表示吗？够不够大规模？

综合：方向有潜力，需要 (1) 具体数值预测+实验验证 (2) 探索模型揭示的物理规律
```

## 示例：对比分析

**输入：** "古早大师 vs 诺奖级 怎么看 transformer 架构？"

| 维度 | 古早大师 | 诺奖级 |
|------|---------|--------|
| 核心关切 | 是否揭示不变原理？能否用物理图像理解？ | 是否受大脑启发？能否解决真实问题？ |
| 优点 | 注意力机制有数学优雅性(+), 可以计算(+) | 学到的表示很强大(+), 大规模实用(+) |
| 缺点 | 参数太多(-), 不理解因果(-) | 缺乏可解释性(-), 能耗太大(-) |

## 快速开始

**快速模式（零配置）：**
```bash
# 拷贝SKILL.md到你的Claude Code项目
mkdir -p .claude/skills/scientist-taste/
cp .claude/skills/scientist-taste/SKILL.md your-project/.claude/skills/scientist-taste/
# 然后直接问Claude："召开品味评审会讨论我的xxx项目"
```

**完整模式：**
```bash
git clone https://github.com/ezy1999/Scientist-Skill.git
cd Scientist-Skill && pip install -e ".[dev]"
scientist-taste list         # 查看所有科学家画像
scientist-taste evaluate "your theory" -s feynman
scientist-taste group "your theory" -l classical
```
