## 为什么需要这个？

当你面对一个科研问题，你可以只依赖自己的判断。但想象一下，如果你能召集一个跨越 400 年的科学家顾问团——让牛顿审视你的数学严格性，让费曼检验你的物理图像，让卡里科提醒你坚持的价值，让辛顿评估你的架构选择——你的研究决策质量会有多大提升？

**这就是 Scientist-Skill 做的事。**

它不是角色扮演。它从每位科学家的论文、书信、演讲、传记中提取他们的**品味轴**（taste axes）——即他们评判"什么是好研究"的显性和隐性标准——然后将这些标准应用到你的问题上。

## 四种使用模式

### 模式1：单人品味
> "用费曼的品味评估我的方法"

系统先给出该科学家品味视角的**深度自然语言分析**，再给出轴评分。

### 模式2：层级选择
> "用诺奖级科学家的标准看看这个方向"

选择一个层级（古早/诺奖/当代PI），系统用该层级全部科学家的品味综合评估。

### 模式3：群体讨论（最强大的模式）
> "召开品味评审会来讨论我的项目"

每位科学家各自发言，观点冲突会被突出，最后给出综合评审和行动建议。

### 模式4：对比分析
> "古早大师 vs 诺奖级，怎么看我的研究方向？"

以表格形式展示不同层级/个人对同一问题的评估差异。

## 完整示例：群体讨论模式

**用户：** "我想用大模型自动发现药物靶点，方法是：用 GPT-4 阅读大量文献，提取基因-疾病关系，构建知识图谱，然后用图推理发现新的靶点。召开品味评审会。"

---

**Einstein 视角：**

你的方案本质上是信息整合，不是规律发现。真正有价值的问题是：基因-疾病关系背后是否存在某种统一的数学结构？如果所有靶点都可以从某个几何框架中推导出来——比如蛋白质相互作用网络的某种拓扑不变量——那就触及了深层原理。但如果只是从A论文发现基因X和疾病Y的相关性，从B论文发现Y和Z的关系，拼出X→Z——这只是信息传递，不是科学发现。

建议：问问你自己——如果你的系统发现了一个新靶点，你能用一个简洁的原理解释**为什么**它是靶点吗？

```
  unity         +0.30 — 知识图谱有统一框架的潜力，但当前设计是碎片化的
  simplicity    -0.20 — GPT-4 + 知识图谱 + 图推理，层次太多
  invariance    +0.10 — 靶点应该与文献来源无关（不变性）
  empirical     +0.50 — 可以用实验验证预测的靶点
```

**Feynman 视角：**

停。先回答一个问题：你的系统有没有重新发现一个**已知的**靶点？

在你声称能发现"新"靶点之前，你得先证明你的系统能独立地重新发现PD-1、HER2或者任何一个已经被验证的靶点。如果做不到——你凭什么相信它发现的"新"靶点是真的，而不是cargo cult？

而且，"用GPT-4阅读文献"——你真的理解GPT-4在做什么吗？你能画一张图说明从文本到靶点的每一步在物理上代表什么吗？如果不能，你在使用一个你自己不理解的黑盒来做科学发现。这不是好科学。

```
  computational_pragmatism +0.30 — 至少产出了可验证的靶点预测
  empirical_ruthlessness   -0.30 — 没看到重新发现已知靶点的验证
  physical_intuition       -0.50 — 从文本到靶点缺少物理/生物学直觉
  anti_formalism           +0.20 — 没有过度数学化，这倒不错
  independent_thinking     -0.40 — GPT-4做发现等于外包思考
```

**Mendel 视角：**

你的数据质量控制在哪里？文献中的基因-疾病关系很多是小样本研究、未重复验证的结果。你把这些低质量数据喂给模型，模型输出的也是低质量结论——garbage in, garbage out。

我的建议：筛选只使用样本量>1000、被独立重复过的基因-疾病关联。然后看你的知识图谱和大样本图谱的一致性有多高。

```
  quantitative_precision +0.20 — 有定量输出
  controlled_experiment  -0.40 — 没有控制组：随机生成的图谱能发现多少"靶点"？
  patient_accumulation   -0.30 — 需要验证大量预测才有统计意义
  pattern_recognition    +0.40 — 知识图谱本身是模式识别工具
```

**Hinton 视角：**

你的方向对了一半——用学到的表示（而非手工规则）来理解生物系统。但你的架构选择有问题。为什么是"先提取关系再构建图谱再推理"这种串行管道？AlphaFold 的教训是：最好的架构是把领域知识直接嵌入到端到端学习中。

如果我来做，我会用一个大的生物医学预训练模型，直接从原始文献+基因组数据端到端学习靶点预测，而不是中间加那么多手工步骤。每个手工步骤都是信息瓶颈。

```
  representational_learning +0.30 — 知识图谱是一种表示，但不够端到端
  architecture_simplicity   -0.30 — 管道太复杂，应该端到端
  biological_inspiration    +0.20 — 基因网络确实有生物基础
  empirical_scaling         +0.40 — 数据量够大的话可能work
```

**Karikó 视角：**

有多少人在做类似的事？如果很多人都在用LLM做drug discovery——那你的竞争优势在哪里？我的经验是：最好的工作来自别人不愿意做、看不上的方向。如果你做这个只是因为LLM热门，那要小心。

但如果你有一个别人没有的数据源，或者你关注的是一个被忽略的疾病领域——那就值得坚持。mRNA被忽视了30年，最终证明了自己。

另外：你有初步结果吗？哪怕验证了一个已知靶点？一个具体的成功案例胜过一百页方案。

```
  experimental_persistence +0.20 — 还在方案阶段，需要更多坚持
  technical_precision      -0.20 — GPT-4提取的关系精度存疑
  problem_focused          +0.40 — 靶点发现是真实的临床需求
  underdog_resilience      -0.10 — 这不是冷门方向，竞争激烈
```

---

**综合评审**

| 科学家 | 评分 | 核心一句话 |
|--------|------|----------|
| Einstein | +0.12 | 信息整合≠规律发现，缺乏统一原理 |
| Feynman | -0.08 | 先重新发现一个已知靶点再说 |
| Mendel | -0.02 | 数据质量控制和随机基线在哪里？ |
| Hinton | +0.15 | 方向对，但应该端到端，不要管道 |
| Karikó | +0.06 | 竞争优势在哪？先拿出一个具体成功案例 |

**群体共识：** 方向有潜力但当前设计有显著缺陷。**三个必须回答的问题：**
1. **验证**：能不能重新发现已知靶点？（Feynman的酸性测试）
2. **数据质量**：用什么标准筛选文献中的关系？随机基线对比是什么？（Mendel的控制）
3. **竞争优势**：为什么是你来做这件事？你有什么别人没有的？（Karikó的逆境智慧）

**改进方向：**
- 加入重新发现实验（rediscovery test）作为第一个里程碑
- 考虑端到端架构替代串行管道（Hinton的建议）
- 寻找一个被忽视的疾病领域作为差异化切入点

**用户：** "古早大师 vs 诺奖级 vs 当代PI，怎么看'用AI写学术论文'？"

| 维度 | 古早大师 | 诺奖级 | 当代PI |
|------|---------|--------|--------|
| **核心关切** | 这是理解还是模仿？(Einstein) 你真懂你写的东西吗？(Feynman) | 能不能学到好的研究表示？(Hinton) 写出来的能重复吗？(Karikó) | 效率提升有多大？(Applied Innovator) 方法论严谨吗？(Witten) |
| **支持** | 如果AI帮你发现论文间的统一联系(Einstein+) | 如果AI加速了从假设到验证的循环(Yamanaka+) | 如果AI让你腾出时间做真正的思考(Olah+) |

| **建议** | 用AI辅助思考，不要用它替代思考(Feynman) | 建立明确的人工审核流程(Doudna) | 在Methods和Results中特别谨慎(Witten) |

## 设计理念

**品味超越时代。** 牛顿不知道量子计算，但他对数学严格性的要求可以评估任何量子算法论文。费曼不知道LLM，但他的Cargo Cult检验可以识别任何领域的伪科学。

**对话优先，评分辅助。** 系统先给出每位科学家视角的自然语言深度分析（科研建议、假设生成、方法论批评），然后才是品味轴的量化评分。目标是对话和启发，不是打分排名。

**证据驱动，非角色扮演。** 每位科学家的品味轴都可追溯到具体的历史证据（论文、书信、演讲、传记分析）。系统不是在模仿他们说话，而是在应用他们有据可查的评判标准。
